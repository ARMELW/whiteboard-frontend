Purpose

This test verifies that shape elements are correctly rendered in scene thumbnails with both camelCase and snake_case property naming conventions.

Issue: Shape elements were not appearing in scene preview thumbnails because the sceneExporter was looking for snake_case properties (shape_type, fill_color) but the actual data used camelCase (shape, fill, stroke).
Fix: Updated renderShapeLayer function to support both naming conventions for backward compatibility.

Test Data Structure

CamelCase Properties (Current Format)

{
  id: 'shape-layer-1',
  type: 'shape',
  position: { x: 500, y: 300 },
  shape_config: {
    shape: 'rectangle',      // camelCase
    fill: '#FF0000',         // camelCase
    stroke: '#000000',       // camelCase
    strokeWidth: 3,          // camelCase
    width: 200,
    height: 100,
    fillMode: 'both'
  }
}

Snake_Case Properties (Backward Compatibility)

{
  id: 'shape-layer-1',
  type: 'shape',
  position: { x: 500, y: 300 },
  shape_config: {
    shape_type: 'rectangle',  // snake_case
    fill_color: '#FF0000',    // snake_case
    stroke_color: '#000000',  // snake_case
    stroke_width: 3,          // snake_case
    width: 200,
    height: 100,
    fill_mode: 'both'
  }
}
